# Production Discount — Blockers vs Demo

Список зон, где **production web** по текущей реализации может восприниматься **слабее demo** или не соответствует заявленной строке матрицы Wave 1. Пока пункт открыт — **нельзя заявлять полный UI parity Wave 1** без approved exception по [DEMO_TO_WEB_ALIGNMENT_GUIDE.md](./DEMO_TO_WEB_ALIGNMENT_GUIDE.md).

---

## Открытые блокеры (верификация кода / UX)

| ID | Зона | Матрица / маршрут | Проблема относительно demo | Тип |
|----|------|-------------------|----------------------------|-----|
| B-01 | Client detail | W1-SALES-DETAIL · `/app/sales/[id]` | Реализация на **generic** карточках (`rounded-xl border` + секции), а не композиция **ClientCard** / воронки как в `src/modules/sales/` + связанные виджеты | visual + UX |
| B-02 | SKU detail | `/app/catalog/[sku]` | Нет выравнивания на **sku-detail** / плотность паттернов demo; преимущественно «админские» карточки вместо эталонной карточки товара | visual + UX |
| B-03 | Content asset | W1-CONTENT-ID · `/app/content/[id]` | Упрощённая страница ассета vs медиа-деталь / сетка в **ContentCenter** | visual + UX |
| B-04 | Автоматическая регрессия | Playwright | Baseline скриншоты привязаны к **платформе** (`win32` / `linux` в имени файла); CI на другой ОС без своих снапшотов или без update — **ложные или пропущенные** диффы | process |

---

## Не блокеры parity с demo (но обязательны к явной маркировке)

| Зона | Пояснение |
|------|-----------|
| Маршруты без demo-экрана | Статус `n_a` в матрице; требование — консистентность **shell** + токены |
| Login mock | В demo может не быть; эталон уточнять; текущий `/login` — осознанный mock |
| Каталог как модуль | В матрице уже допускается `n_a` для «модуля каталога» в demo; **список** web всё равно должен быть консистентен с дизайн-системой |

## Закрытие блокера

1. Реализация, соответствующая demo, **или**  
2. Запись в матрице: `exception` + `exception_status=approved` + `rationale_if_exception` (1 из 4 оснований Alignment Guide) + владелец продукта.

После закрытия B-01…B-03 для Wave 1 обновить:

- строки матрицы → `parity`;  
- при необходимости `npm run test:e2e:update` для новых композиций;  
- [PARITY_CHECKLIST_FINAL.md](./PARITY_CHECKLIST_FINAL.md) § A.
