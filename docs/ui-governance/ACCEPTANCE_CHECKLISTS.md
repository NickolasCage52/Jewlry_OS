# Acceptance Checklists — Design / Front-end / QA

Используются при приёмке работ по **Demo ↔ Web parity**. Закрытие чеклиста — в тикете или PR.

Для **финальной приёмки (Stage 5)** используйте расширенные артефакты: [QA_ACCEPTANCE_LAYER.md](./QA_ACCEPTANCE_LAYER.md), [PARITY_CHECKLIST_FINAL.md](./PARITY_CHECKLIST_FINAL.md), [STAGE_5_QUALITY_GATE_LAYER.md](./STAGE_5_QUALITY_GATE_LAYER.md).

## Design — перед передачей в разработку

- [ ] Эталон demo зафиксирован (путь к модулю + при необходимости скрин/коммит).
- [ ] Для каждого экрана scope перечислены состояния (включая edge cases из матрицы).
- [ ] Сетка/spacing/typography/color согласованы с `styles/variables.css` и `components.css`.
- [ ] Иконки, карточки, кнопки, поля — правила состояний описаны или показаны на эталоне.
- [ ] Навигация: активные состояния, группировка, соответствие смыслу demo sidebar.
- [ ] Микроанимации: параметры или явное «как в demo / отсутствует в demo».
- [ ] Адаптив: правила stack/collapse; нет скрытия сценария без альтернативы.
- [ ] Список **антипаттернов**: не менять композицию «для удобства вёрстки»; не ослаблять иерархию; не подменять demo generic-компонентами.

## Front-end — перед сдачей (PR)

- [ ] Сверка с demo на эталонном вьюпорте для каждого маршрута scope.
- [ ] Токены: используются согласованные переменные/классы; нет дрейфа «магических» отступов (после Stage 2 — только токены).
- [ ] Типографика: семейства, кегли, веса, цвета текста соответствуют спецификации/demo.
- [ ] Радиусы, бордеры, тени, иконки — как в эталоне.
- [ ] Состояния интерактивных элементов полные для scope.
- [ ] Формы: ошибки, disabled, длинные подписи не ломают композицию.
- [ ] Фокус: видимый `focus-visible`; модалки без ловушек фокуса.
- [ ] Доступность: минимум соответствия целевым требованиям без визуальной «дешёвки».
- [ ] Адаптив: проверены брейкпоинты из viewport matrix.
- [ ] Нет заметного layout shift на типовых действиях scope.
- [ ] Анимации не хуже ощущения от demo (плавность/тайминг).

## QA — финальная приёмка

- [ ] Визуал: side-by-side или наложение с demo для ключевых экранов scope.
- [ ] Логика: сценарии demo воспроизводимы на web с теми же ожиданиями.
- [ ] Состояния: empty, loading, error, no access — по матрице.
- [ ] Таблицы/списки: длинный контент, прокрутка, читаемость.
- [ ] Фильтры и поиск: сброс, активное состояние, сохранение контекста (как в эталоне).
- [ ] Модалки: открытие/закрытие, ESC, фокус, скролл фона.
- [ ] Консистентность терминов и паттернов между экранами.
- [ ] Общее восприятие: **web не слабее demo** (обязательная отметка в отчёте).
