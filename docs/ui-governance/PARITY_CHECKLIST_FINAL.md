# Final Parity Checklist — Production-ready (Demo ↔ Web)

Использовать для **финальной приёмки** и периодического аудита. Отметка «OK» только при явном подтверждении (тикет/PR/строка матрицы). Источник правды по визуалу: **demo** (`index.html`, `src/`, `styles/`).

---

## A. Screens (Wave 1 + заявленный scope)

- [ ] **App shell** — sidebar, топбар, crumb, поиск (видимость по брейкпоинтам), аватар, колокол; активный пункт навигации.
- [ ] **Mobile nav** — drawer: открытие, закрытие (backdrop, ESC, переход по ссылке), нет дубля бренда там, где не положено; scroll lock.
- [ ] **Dashboard** (`/app/dashboard`) — иерархия блоков, KPI, сетка, лента/alerts при наличии в эталоне.
- [ ] **Sales list** (`/app/sales`) — двухколоночный сценарий, фильтры, список, превью/empty state.
- [ ] **Sales detail** (`/app/sales/[id]`) — соответствие **demo ClientCard** / воронке (см. blockers, если ещё generic layout).
- [ ] **Content center** (`/app/content`) — сетка/карточки, фильтры, empty.
- [ ] **Content asset** (`/app/content/[id]`) — медиа-деталь vs demo **ContentCenter** / медиа-сетка.
- [ ] **Catalog list** (`/app/catalog`) — таблица/панель в стиле эталонных классов.
- [ ] **Catalog SKU** (`/app/catalog/[sku]`) — **sku-detail** parity vs demo (см. blockers при generic карточках).
- [ ] **Login** (`/login`) — mock-вход, ошибка роли при необходимости.
- [ ] Прочие маршруты scope: либо `n_a` в матрице, либо консистентность shell + токены.

---

## B. Components (пересекающиеся)

- [ ] Кнопки `.btn` / `Button` — primary/secondary, disabled, loading, hover/active/focus-visible.
- [ ] Поля ввода, селекты, поиск — состояния, длинный текст, error.
- [ ] Карточки `.panel` / `.ui-card` — отступы, тени, заголовки.
- [ ] Бейджи статусов сделки/склада — те же семантические цвета/классы, что в demo.
- [ ] Таблицы — header, row hover, длинный контент, горизонтальный overflow контролируемый.
- [ ] Модалки — открытие/закрытие, ESC, фокус на закрытие, **скролл фона заблокирован**, прокрутка тела модалки.
- [ ] Chips / фильтры — активное состояние, сброс.

---

## C. States (критичные)

Для каждой строки матрицы в scope минимум:

- [ ] default / happy path  
- [ ] **hover**, **active**, **focus-visible** на интерактивах  
- [ ] **disabled** / **loading** (кнопки, формы)  
- [ ] **empty** (списки, дашборд при отсутствии данных)  
- [ ] **error** (формы, сегмент `error.tsx`)  
- [ ] **no access** (роль без права — редирект/сообщение по правилам продукта)  
- [ ] **long content** / **overflow** (таблицы, карточки, модалки)  
- [ ] **async update** (после действия UI обновляется предсказуемо, без «прыжков»)

---

## D. Flows (сценарии)

- [ ] Навигация: shell → раздел → деталь → назад; crumb правдив.
- [ ] Sales: фильтр → выбор клиента → деталь (и обратно) без потери контекста эталона.
- [ ] Content: открытие элемента, связь с каталогом если есть в UX demo.
- [ ] Резерв/цена/операции (если в scope): обратная связь видима, нет необоснованного CLS.

---

## E. Adaptive

- [ ] **Desktop эталон** 1440×900 — нет деградации иерархии vs demo.
- [ ] **1280×800** — ритм и первичное действие на месте.
- [ ] **768×1024** — перестроение (stack/collapse/drawer) без потери сценария.
- [ ] **390×844** — мобильный drawer, топбар, нет неконтролируемого горизонтального скролла страницы.
- [ ] Перестройки только как **stack / collapse / drawer / controlled regrouping**; primary action доступен.

---

## F. Consistency

- [ ] Типографика: Cormorant + Jost (как в `layout` web), кегли и веса не противоречат токенам/эталону.
- [ ] Цвета и бордеры через токены; нет «случайных» hex в новых участках scope.
- [ ] Одинаковые паттерны (карточка, таблица, модалка) ведут себя одинаково между экранами.
- [ ] Термины в UI согласованы с demo/глоссарием.

---

## G. Perceived quality (обязательный субъективный барьер)

- [ ] Анимации и переходы **не хуже** ощущения demo (открытие панелей, hover, focus).
- [ ] Нет «сырого продакшена»: рывки скролла, мигание при загрузке, слабый backdrop, неотзывчивые контролы.
- [ ] Уполномоченный ревьюер фиксирует: **web не воспринимается слабее, дешевле или шумнее demo** при прямом сравнении на эталонном и мобильном вьюпорте.
