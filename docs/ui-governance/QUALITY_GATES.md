# Quality Gates — Demo ↔ Web Parity

Gates **обязательны** для работ, затрагивающих UI/UX относительно demo-эталона. Блокировка следующего этапа/релиза при невыполнении указана явно.

## Gate D — Design

**Когда:** перед передачей спецификации в разработку (или перед стартом Stage 2 для токенов).

- [ ] Для целевых строк матрицы Wave N заданы эталонные ссылки на demo (файл/скрин/коммит).
- [ ] Описаны обязательные **состояния** (включая empty / loading / error / focus-visible где применимо).
- [ ] Любое отличие от demo имеет статус `exception` + `approved` + rationale по одному из 4 оснований ([DEMO_TO_WEB_ALIGNMENT_GUIDE.md](./DEMO_TO_WEB_ALIGNMENT_GUIDE.md)).
- [ ] Правила адаптива задокументированы (без потери иерархии и сценария).

**Блок:** старт FE-работ по паритету без закрытого Gate D для соответствующих строк матрицы.

## Gate F — Front-end

**Когда:** перед code review / мержем PR, затрагивающего UI.

- [ ] Реализация сопоставлена с **demo**, не только с локальным вкусом или макетом.
- [ ] Отступы/типографика/цвета через **токены** (после внедрения единой системы в Stage 2); до внедрения — явный список расхождений в PR.
- [ ] Реализованы состояния из матрицы для затронутых компонентов.
- [ ] Нет необоснованного layout shift на ключевых действиях.
- [ ] `focus-visible` и модалки не ломают доступность и не противоречат эталону поведения.

**Блок:** мерж PR при открытых задачах паритета без exception для затронутых экранов.

## Gate Q — QA

**Когда:** перед пометкой релиз-кандидата / закрытием эпика паритета.

- [ ] Пройдены строки матрицы Wave N для целевых маршрутов: статус `parity` или `exception approved`.
- [ ] Пройден [ACCEPTANCE_CHECKLISTS.md](./ACCEPTANCE_CHECKLISTS.md) для затронутых зон.
- [ ] Выполнен **субъективный критерий**: web не воспринимается слабее, дешевле или шумнее demo при сравнении.
- [ ] Адаптив проверен по **viewport matrix** ([VISUAL_REGRESSION_MINIMUM.md](./VISUAL_REGRESSION_MINIMUM.md)).

**Блок:** релиз функционала, заявленного как «parity», без Gate Q.

## Gate R — Release (продакт / ответственный за качество)

**Когда:** финальная приёмка.

- [ ] Закрыты Gate D, F, Q для заявленного scope.
- [ ] Нет незафиксированных расхождений demo / production.
- [ ] Артефакты сверки (скриншоты или отчёт регрессии) приложены к релизу или тикету.
- [ ] [DEFINITION_OF_DONE_UI_PARITY.md](./DEFINITION_OF_DONE_UI_PARITY.md) выполнен для scope (включая **§7 Stage 5** при финальной приёмке parity-волны).
- [ ] [PARITY_CHECKLIST_FINAL.md](./PARITY_CHECKLIST_FINAL.md) и [QA_ACCEPTANCE_LAYER.md](./QA_ACCEPTANCE_LAYER.md) закрыты или исключения approved.
- [ ] Открытые пункты [PRODUCTION_DISCOUNT_BLOCKERS.md](./PRODUCTION_DISCOUNT_BLOCKERS.md) согласованы: либо закрыты, либо оформлены как exceptions.
- [ ] При использовании автоматизации: `npm run test:e2e` в `web/` — успешный прогон **или** зафиксированный эквивалент из [REGRESSION_BASELINE_AND_PROCESS.md](./REGRESSION_BASELINE_AND_PROCESS.md).

**Блок:** продвижение в production при любом «известном» визуальном уступании demo без approved exception.
